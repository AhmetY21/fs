## 2024-05-24 - Rate Limit IP Spoofing
**Vulnerability:** The rate limiter in `app/api/analyze/route.js` solely relied on `request.headers.get('x-forwarded-for')` to extract the client IP. This header can be easily spoofed by malicious clients, allowing them to bypass rate limiting completely by changing the header value on each request.
**Learning:** `x-forwarded-for` is not a secure or reliable source for client IPs unless explicitly verified by a trusted reverse proxy. Next.js provides `request.ip` for this purpose (when deployed on platforms like Vercel), and `x-real-ip` is often injected securely by proxies. We must prioritize these secure sources before falling back to `x-forwarded-for`. Removing `x-forwarded-for` entirely is risky as it might be the only source in some non-Vercel environments, leading to global rate-limiting (a DoS risk).
**Prevention:** Always prioritize `request.ip` and `request.headers.get('x-real-ip')` when extracting client IPs for rate limiting. Use `x-forwarded-for` only as a fallback, ensuring to parse and extract the first IP correctly (`.split(',')[0].trim()`).
