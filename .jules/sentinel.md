## 2026-03-22 - Add missing security headers
**Vulnerability:** The application was missing critical security headers like Content-Security-Policy, Strict-Transport-Security, X-Frame-Options, X-Content-Type-Options, Referrer-Policy. This lack of security headers left the application vulnerable to common attacks such as Cross-Site Scripting (XSS), Clickjacking, and MIME sniffing.
**Learning:** Security headers should be explicitly configured in Next.js applications using `next.config.mjs` or `next.config.js` via the `headers()` function, as Next.js doesn't inject these by default. We must also consider external domains in our Content Security Policy, such as https://generativelanguage.googleapis.com, to prevent our API requests from being blocked by the browser.
**Prevention:** Make sure to always configure a basic set of recommended security headers during the initial setup of a Next.js application, including CSP rules tailored to the application's external connections.
