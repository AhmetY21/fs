## 2024-05-24 - Missing CSRF Validation in API Route
**Vulnerability:** The `/api/analyze` Next.js App Router API route did not validate the Origin or Referer header, making it vulnerable to Cross-Site Request Forgery (CSRF). While the frontend uses standard Fetch with content-type `application/json`, which normally triggers a preflight, relying solely on this behavior or CORS configurations outside the route logic isn't sufficient defense-in-depth, especially if non-browser clients can interact with it.
**Learning:** Next.js App Router API routes must explicitly implement CSRF validation by checking the `Origin` or `Referer` headers against the expected `Host` or `x-forwarded-host`, as this protection is not built-in by default.
**Prevention:** Always implement explicit header validation (comparing Origin/Referer to Host) in state-changing (POST/PUT/DELETE) API routes. To support non-browser clients, fail-open when both headers are completely absent, but strictly validate them when present. Use `new URL()` within a `try-catch` block to safely parse origin strings.
