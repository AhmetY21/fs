## 2025-02-18 - Missing CSRF Protection & Insecure IP Extraction in API Route
**Vulnerability:** The `/api/analyze` Next.js endpoint lacked CSRF protection, and its rate limiter relied solely on `x-forwarded-for` to extract IPs, which can be spoofed.
**Learning:** Next.js API routes do not have built-in CSRF protection by default. We must explicitly validate the `Origin` header against the `Host` or `x-forwarded-host` headers to ensure requests originate from our frontend. However, strictly enforcing this by failing closed when headers are missing can break non-browser API usage. Additionally, extracting client IPs should prioritize platform-provided properties like `request.ip` or `x-real-ip` before falling back to properly parsed `x-forwarded-for` strings, as removing the fallback entirely risks global rate limiting and DoS in environments where primary IP properties are missing.
**Prevention:** Implement Origin vs Host validation for state-changing or sensitive API routes in Next.js, but fail open if the Origin is not present to support non-browser clients unless the route is strictly browser-only. Use a comprehensive IP extraction utility that prioritizes secure headers and correctly parses comma-separated proxy lists as a fallback to avoid global rate limiting.
