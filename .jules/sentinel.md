## 2024-03-30 - Fix CSRF missing protection and secure IP extraction in shared APIs
**Vulnerability:** The API endpoint `app/api/analyze/route.js` was missing Cross-Site Request Forgery (CSRF) protection. Additionally, the IP extraction for rate limiting incorrectly relied purely on the `x-forwarded-for` header without safely parsing proxy modifications.
**Learning:** For Next.js backend API routes shared with mobile apps like Flutter, standard security headers (Origin, Host) may not be sent automatically. Adding CSRF protection requires a graceful fallback to skip checks if these headers are completely missing, so as not to break non-browser API clients.
**Prevention:** Always implement standard CSRF header validations (comparing Origin to Host / x-forwarded-host) on API routes, wrapped in try-catch to handle malformed URLs. To reliably extract client IP addresses across environments, prioritize `request.ip` and `x-real-ip` before safely parsing the first element of `x-forwarded-for`.
