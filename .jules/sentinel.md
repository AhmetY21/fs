## 2025-04-14 - Missing CSRF Protection in API Route
**Vulnerability:** The `/api/analyze` Next.js route handler lacked Cross-Site Request Forgery (CSRF) protection. Malicious websites could trick authenticated users' browsers into issuing state-changing or quota-consuming POST requests on their behalf, as standard browser behavior automatically includes credentials (if any) and lacks origin enforcement.
**Learning:** Next.js App Router API routes (`app/api/*/route.js`) are inherently stateless but do not provide built-in CSRF protection by default, especially for endpoints accepting `POST` requests. We must manually validate `Origin` and `Referer` against the `Host` or `x-forwarded-host` headers to ensure requests originate from the expected domain.
**Prevention:** Always implement header validation for origin or referer on state-changing API endpoints or ones that consume substantial resources (like AI API calls). Gracefully fail-open if the headers are missing to not break non-browser API clients, but strictly validate them if present.
