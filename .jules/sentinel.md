## 2024-03-31 - Fix CSRF missing validation with x-forwarded-host
**Vulnerability:** The API endpoint `app/api/analyze/route.js` lacked CSRF protection, failing to validate `origin` against the host, making it susceptible to cross-site request forgery attacks.
**Learning:** Next.js applications deployed behind multi-tier proxies often require validation against the `x-forwarded-host` header rather than just `host`. Furthermore, shared backend architectures (e.g., serving both browsers and Flutter mobile apps) must bypass checks if the `origin` header is entirely absent to gracefully support non-browser clients which typically do not send security headers like `origin`. Additionally, extracting the primary host from `x-forwarded-host` requires splitting the header into an array and pulling the first string.
**Prevention:** In shared backend APIs, validate `origin` strictly against `host` or `x-forwarded-host` `.split(',')[0].trim()` when the `origin` header is present, while safely parsing the `origin` using `new URL()` in a `try-catch` block.
