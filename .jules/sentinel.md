## 2024-05-02 - Rate Limit IP Spoofing Bypass
**Vulnerability:** The rate limiter in `app/api/analyze/route.js` extracted the IP address using `(request.headers.get('x-forwarded-for') ?? '127.0.0.1').split(',')[0].trim()`. This left the application vulnerable to IP spoofing bypass, as an attacker could freely manipulate the leftmost IP in the `X-Forwarded-For` chain. By sending multiple requests with different spoofed IPs prepended to the `X-Forwarded-For` header, they could bypass the 10-request limit.
**Learning:** In Next.js applications and generic reverse proxy setups, the `X-Forwarded-For` header is constructed by appending IPs from left to right. The leftmost IP is the original client IP, but it can be easily spoofed if the client sends a custom header. The rightmost IP is appended by the last trusted proxy in the chain, making it the most reliable indicator of the true client IP if `request.ip` or `x-real-ip` are unavailable.
**Prevention:** When extracting client IPs for security mechanisms (like rate limiting), prioritize `request.ip` or `x-real-ip`. If relying on `X-Forwarded-For`, always use the last IP in the chain (e.g., `forwardedFor.split(',').pop()`) to prevent spoofing, rather than the first.
