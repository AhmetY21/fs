## 2024-05-24 - CSRF and IP Spoofing Prevention in Next.js API Routes
**Vulnerability:** The application's `/api/analyze` endpoint was vulnerable to IP spoofing because it implicitly trusted the `x-forwarded-for` header for rate-limiting, and it lacked CSRF protection, allowing cross-site browser requests.
**Learning:** In Next.js App Router API handlers, explicitly relying on `x-forwarded-for` can lead to trivial IP spoofing if not properly validated behind a trusted proxy. Furthermore, Next.js API routes do not automatically enforce CSRF protection; it must be implemented manually by checking headers like `Origin` and `Host`.
**Prevention:** Always use `request.ip` (provided by Next.js `NextRequest`) as the primary source for the client's IP address, falling back to `x-forwarded-for` only for local development scenarios. For CSRF protection, validate that the `Origin` header matches the `Host` header, using `new URL(origin)` within a `try-catch` block to handle malformed strings gracefully.
