## 2024-03-22 - CSRF Protection in Next.js API Routes
**Vulnerability:** Missing CSRF validation in sensitive Next.js API routes allowed for potential cross-site request forgery attacks if the API is accessed by browsers from arbitrary origins.
**Learning:** Next.js Route Handlers (API routes) do not have built-in CSRF protection like Server Actions. The `Origin` and `Referer` headers must be explicitly validated against the `Host` or `X-Forwarded-Host` headers to prevent malicious cross-origin requests. It's crucial to gracefully fail-open if the headers are completely missing to avoid breaking non-browser API clients, but strictly validate them against the host if they are present. Ensure `Origin` is safely parsed using `new URL()` within a try-catch block.
**Prevention:** Always implement header validation for origin/referer against host in state-modifying or sensitive Next.js API routes that might be accessed from a browser context, avoiding reliance on rate-limiting alone for API security.
